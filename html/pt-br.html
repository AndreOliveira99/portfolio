Desenvolvimento

Ainda estou a procura da minha primeira experiência full time "oficial" como programador. No entando, na minha jornada profissional até aqui tive a oportunidade de solucionar alguns problemas reais utilizando programação:

  • Ainda quando estagiário na SolarView, houve uma espécie de hackathon para solucionar um problema de detecção de sombreamento nas curvas de energia solar. Resumindo a história, eu apresentei uma ideia para solucionar, que foi na ocasião melhor candidata à solucionar o problema. Porém para testá-la, era necessário um trabalho manual enorme com os dados usando planilhas. Então, criei um programa usando o octave (uma espécie de matlab paralelo) para automatizar o trabalho manual com os dados, o que tornou as análises ao menos 10x mais rápidas.
  • Quando supervisor comercial, nossos leads vinham em sua maior parte de sites que listavam integradores, porém com o tempo começou a surgir um problema: entravamos em contato com muitos leads todas as semanas, então o trabalho de copiar os dados que esses sites forneciam para uma planilha (e posteriormente um CRM de vendas), passou a ser gigantesco. Então, um desenvolvedor da empresa me ensinou a técnica do web scrapping, para criar um bot que pudesse coletar os dados de praticamente qualquer site encontrado de forma automatizada. A partir de então, passei a criar bots para coletar leads de diversos sites de forma automatizada, para utilizá-los em nossa força de vendas.
  • Além disso, agora que eu havia voltado a praticar a programação, comecei a resolver outros problemas utilizando código, como por exemplo, o de verificar se os novos leads já tinham sido contatados, e também verificar se a empresa ainda se o CNPJ da empresa ainda estava ativo.

Jogo da memória

Esse foi o meu primeiro projeto utilizando html, css e javascript. A ideia do jogo veio de um vídeo do canal "freecodecamp.org". Esse canal possuí um tutorial sobre como construir o jogo, então eu decidi contruir primeiramente do meu jeito, e depois comparar o meu código com a solução do vídeo, para melhor aprendizado. O projeto é um jogo da memória clássico, e deve funcionar da seguinte forma:

  • O jogo possui uma série de cartas, e cada carta possui um, e apenas um par idêntico
  • As cartas possuem imagens únicas em sua face (com um par idêntico cada). Todas as cartas tem o verso idêntico.
  • O jogo inicia com todas as cartas posicionadas viradas para baixo, de forma que o jogador não pode ver o conteúdo. As cartas são embaralhadas em ordem aleatória e posicionadas lado a lado
  • O jogador pode escolher duas cartas viradas para baixo, revelando sua face: se as duas cartas tem a face idêntica, um ponto é marcado e as duas cartas permanecem viradas para cima até o final do jogo. Se as duas cartas tem faces diferentes, as cartas são viradas para baixo novamente
  • O jogo acaba quando todos os pares idênticos são descobertos, e todas as cartas estão viradas para cima

Como esse foi o meu primeiro projeto utilizando html e css, há MUITO o que melhorar nele. Eu provavelmente vou recriá-lo em breve.

Flappy Bird

This project was made as a final challenge of a HTML, CSS, JS online course from Udemy, hosted by "cod3r.com". Esse projeto foi o desafio final de um curso online da "cod3r.com", que ensina HTML, CSS e JS. O desafio era basicamente criar um jogo similar ao "flappy bird", usando HTML, CSS e JS. Então, o professor exibiu o funcionamento do jogo pronto, e deu a dica de que deveríamos usar flex-box para criar o jogo. O jogo deve funcionar da seguinte forma:

  • O pássaro (representado por um quadrado amarelo), deve voar através dos canos posicionados no topo e na base da tela.
  • Os canos devem possuir tamanhos aleatórios, mas o espaço entre eles deve ser sempre o mesmo
  • O pássaro deve estar sempre caindo e se movendo para a direita com uma velocidade constante
  • O jogador pode pressionar uma tecla para fazer com que o pássaro ganhe altitude
  • Para cada par de canos que o pássaro atravesse sem tocá-los, um ponto é marcado
  • Se o pássaro tocar algum cano, o jogo acaba
  • O jogador pode pressionar uma tecla pra reiniciar o jogo

Jogo das bandeiras

Esse projeto foi feito para praticar minhas habilidades de programação html, javascript e css. O principal objetivo do projeto era entender melhor o funcionamento dos seguintes conceitos:

  • Formulários Html
  • Função construtora
  • Requisições Ajax/Axios
  • Servidores Node

Minha irmã deu a ideia de um jogo no qual você deve acertar as bandeiras de países conforme a imagem das bandeiras. O jogo deve funcionar da seguinte forma:

  • o jogo exibe a bandeira de um país aleatório
  • o jogador deve responder qual é o país a qual a bandeira pertence
  • se o jogador responder corretamente, um ponto é marcado
  • após confirmar a responsta, o jogo deve exibir qual era a alternativa correta, para propósitos educacionais
  • Após finalizar o jogo, o jogador pode enviar sua pontuação para um ranking de maiores pontuações, e ver sua colocação no ranking

Baseado nessas premissas, eu decidi construir o jogo de uma forma similar à um sistema de provas online de uma escola, para que eu possa possívelmente encontrar um uso prático ao código futuramente.
